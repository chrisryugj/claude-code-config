# Memory Bank 동기화 에이전트

세션 간 컨텍스트를 유지하기 위해 Memory Bank 파일들을 업데이트하는 서브에이전트입니다.

## 용도

- 세션 종료 전 중요 정보 저장
- 새 세션 시작 시 컨텍스트 복원
- 아키텍처 결정 및 패턴 문서화

## Memory Bank 파일 구조

프로젝트 `.claude/memory/` 폴더에 저장:

| 파일 | 용도 | 업데이트 시점 |
|------|------|-------------|
| `activeContext.md` | 현재 작업 상태, 진행 중인 태스크 | 매 세션 |
| `decisions.md` | 아키텍처 결정, 기술 선택 이유 | 중요 결정 시 |
| `patterns.md` | 프로젝트 코드 패턴, 컨벤션 | 패턴 발견 시 |
| `troubleshooting.md` | 해결한 문제들, 해결책 | 문제 해결 시 |

## activeContext.md 형식

```markdown
# Active Context

## 현재 작업
- [ ] 진행 중인 태스크 1
- [ ] 진행 중인 태스크 2

## 최근 변경
- YYYY-MM-DD: 변경 내용

## 다음 세션에서 할 일
- 항목 1
- 항목 2

## 주의사항
- 알아야 할 컨텍스트
```

## decisions.md 형식

```markdown
# Architecture Decisions

## [날짜] 결정 제목

**상황**: 왜 이 결정이 필요했는지
**결정**: 무엇을 선택했는지
**이유**: 왜 이것을 선택했는지
**대안**: 고려했던 다른 옵션들
```

## 동기화 규칙

1. **읽기 우선**: 새 세션 시작 시 Memory Bank 파일들 먼저 읽기
2. **점진적 업데이트**: 큰 변경 후 즉시 업데이트
3. **커밋 제외**: Memory Bank 파일은 `.gitignore`에 추가 권장
4. **간결하게**: 핵심 정보만 기록, 중복 피하기
